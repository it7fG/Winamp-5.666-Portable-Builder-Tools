# Winamp 5.666 Portable Builder + Tools

Набор скриптов, который **превращает обычную установку Winamp 5.666 в чистую портативную сборку** и добавляет к ней несколько полезных утилит для работы с плейлистами и альбомами.

> **Важно:** этот репозиторий **НЕ содержит** сам Winamp и не распространяет его файлы. Winamp 5.666 — проприетарное бесплатное ПО (freeware), его бинарники сюда не входят. Здесь только **мой код**: скрипт сборки и утилиты. Winamp ты скачиваешь и устанавливаешь сам, а скрипт собирает из него портативную папку у тебя на компьютере. Подробнее — в разделе [О Winamp и лицензии](#о-winamp-и-лицензии).

---

## Что это и зачем

У классического Winamp 5.666 нет официального портативного режима «из коробки», а настройки он раскидывает по системе (`%APPDATA%`, реестр и т. д.). Этот проект решает две задачи:

1. **`BuildPortable.ps1`** — собирает из твоей установки Winamp **чистую портативную папку**:
   - все настройки и плейлисты лежат **внутри** папки сборки (рядом с `winamp.exe`), а не в профиле Windows;
   - папку можно скопировать на флешку или другой ПК — Winamp сохранит все настройки;
   - из сборки удаляются твои личные данные (пути к музыке, идентификаторы установки, координаты окон, кэши медиатеки);
   - отключается всплывающее окно «Доступна новая версия»;
   - удаляется ненужный для портатива деинсталлятор.

2. **`tools/`** — утилиты, которые кладутся внутрь портативной сборки (и работают также с обычной установкой):
   - массовое добавление альбомов как отдельных плейлистов;
   - массовое удаление плейлистов;
   - блокировка окна обновлений;
   - генерация текстового трек-листа альбома (с битрейтом, длительностью и размером).

## Требования

- **Windows 7 или новее**
- **PowerShell 2.0+** — уже встроен в Windows, ставить ничего не надо
- **Установленный Winamp 5.666** (нужен как источник файлов для сборки)

## Как собрать портативную версию

### Шаг 1. Установи Winamp 5.666

Скачай оригинальный установщик Winamp 5.666 из любого официального/архивного источника и установи как обычно. (Ссылку на скачивание этот репозиторий намеренно не приводит — см. раздел про лицензию.)

### Шаг 2. Положи скрипты в отдельную рабочую папку

Клонируй или распакуй этот репозиторий, например в `C:\WinampBuild\`. Должно получиться:

```
C:\WinampBuild\
├── BuildPortable.ps1
├── BuildPortable.bat
├── tools\
└── README.md
```

### Шаг 3. Скопируй файлы установленного Winamp рядом со скриптом

Скрипту нужны **две папки-источника** с такими именами:

| Создай папку | Скопируй в неё содержимое из |
|--------------|------------------------------|
| `WinampProgram Files\` | `C:\Program Files (x86)\Winamp` |
| `WinampRoaming\` | `%APPDATA%\Winamp` (вставь этот путь в адресную строку Проводника) |

После этого структура такая:

```
C:\WinampBuild\
├── BuildPortable.ps1
├── BuildPortable.bat
├── tools\
├── WinampProgram Files\   ← копия C:\Program Files (x86)\Winamp
└── WinampRoaming\         ← копия %APPDATA%\Winamp
```


### Шаг 4. Запусти сборку

**Просто дважды кликни `BuildPortable.bat`.**

> Почему именно `.bat`, а не `.ps1`? Двойной клик по `.ps1` в Windows по умолчанию **открывает его в редакторе**, а не запускает. `.bat`-обёртка запускает PowerShell правильно (с `-ExecutionPolicy Bypass` только для одного запуска — глобальные настройки системы не меняются).

Альтернатива из PowerShell вручную:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\BuildPortable.ps1"
```

### Шаг 5. Готово

Появится папка:

```
C:\WinampBuild\dist\Winamp_Portable_5.666\
```

Внутри — готовый портативный Winamp: запускай `winamp.exe` (или `Winamp.bat`). Все настройки теперь хранятся в этой папке. Можешь заархивировать её и переносить куда угодно.

### Шаг 6. Убери лишнее (важно!)

После успешной сборки рабочей папки три **исходные** папки больше не нужны — удали их:

```
C:\WinampBuild\
├── BuildPortable.ps1        ← можно удалить
├── BuildPortable.bat        ← можно удалить
├── WinampProgram Files\     ← УДАЛИТЬ
├── WinampRoaming\           ← УДАЛИТЬ
├── tools\                   ← УДАЛИТЬ (исходный!)
└── dist\
    └── Winamp_Portable_5.666\   ← ОСТАВИТЬ только это
```

> ⚠️ **Частая путаница с папкой `tools\`.** Их **две**:
> - `C:\WinampBuild\tools\` — это **исходник** для сборки. Запускать утилиты отсюда **бесполезно** — они не привязаны к портативному плееру.
> - `...\dist\Winamp_Portable_5.666\tools\` — это **рабочие** утилиты внутри собранного плеера. **Запускать нужно отсюда.**
>
> Чтобы не путаться — после сборки удали исходный `tools\` (и две папки-источника), оставив только готовую папку `Winamp_Portable_5.666`.

Итог: оставляешь **только** `Winamp_Portable_5.666\` — это и есть твой готовый портативный Winamp со встроенными утилитами в `tools\` внутри.

## Что именно делает скрипт при сборке

| Шаг | Действие |
|-----|----------|
| 1 | Копирует программные файлы Winamp (без деинсталлятора `unins000.*` и без `paths.ini`) |
| 2 | Копирует пользовательские данные, **вычищая личное**: пути к музыке, `uid`, статистику, координаты окон, кэши медиатеки, отчёты о сбоях |
| 3 | Чистит `winamp.ini` от персональных ключей |
| 4 | Кладёт `winamp.ini` рядом с `winamp.exe` — это включает портативный режим Winamp |
| 5 | Копирует утилиты из `tools\` |
| 6 | Добавляет лаунчер `Winamp.bat`, `README.txt` и блокировку окна обновлений |

## Утилиты (`tools/`)

Подробное описание каждой — в [`tools/README.md`](tools/README.md). Кратко:

| Файл | Что делает |
|------|------------|
| `WinampAddAlbums.bat` | Массово добавляет альбомы как отдельные плейлисты |
| `WinampRemovePlaylists.bat` | Массово удаляет плейлисты (с галочками) |
| `WinampBlockUpdates.bat` / `WinampUnblockUpdates.bat` | Блокировка/откат окна обновлений (правит `hosts`, нужен админ) |
| `MakeTracklist.bat` | Делает текстовый трек-лист альбома + битрейт, длительность, размер |

Все утилиты «portable-aware»: если лежат внутри портативной сборки — пишут в её плейлисты; если рядом обычная установка — в `%APPDATA%\Winamp`.
