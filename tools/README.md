# Winamp Tools

Небольшой набор скриптов для **Winamp 5.x** на Windows. Делает несколько вещей, которые в самом плеере либо не автоматизированы, либо неудобны:

1. **Массово добавляет альбомы в "Списки воспроизведения"** — выбираешь папку с подпапками-альбомами, и каждая подпапка становится **отдельным плейлистом**.
2. **Массово удаляет плейлисты** — все или выборочно, через окно с галочками.
3. **Блокирует всплывающее окно "Доступна новая версия Winamp"**.
4. **Делает текстовый трек-лист альбома** — список песен + битрейт, общая длительность и размер папки.

## Требования

- Windows 7 или новее
- Установленный Winamp (5.x)
- PowerShell 2.0+ — **уже встроен в Windows**, ничего ставить не надо
- Python **не нужен**

## Установка

Распакуй архив (или клонируй репозиторий) в любую папку — например, на рабочий стол. Запуск двойным кликом по нужному `.bat`-файлу.

## 1. Массовое добавление альбомов

**Файл:** `WinampAddAlbums.bat`

### Как использовать

1. Закрой Winamp (важно: при выходе он перезаписывает `playlists.xml`).
2. Запусти `WinampAddAlbums.bat` двойным кликом.
3. В диалоге выбери папку, внутри которой лежат подпапки-альбомы.
4. Дождись окна "Готово" с отчётом.
5. Открой Winamp — новые плейлисты появятся в **"Списки воспроизведения"**.

### Пример

Структура на диске:

```
D:\Music\Christian\
├── Asarela - I Believe (2020)\
│   ├── 1. Asarela - You Never Give up on Me.mp3
│   ├── 2. Asarela - I Am Not Afraid.mp3
│   └── ...
├── Jesus Culture - Consumed (2009)\
│   └── ...
└── New Wine - Awaken Us (2021)\
    └── ...
```

Выбираешь `D:\Music\Christian\` → получаешь **3 отдельных плейлиста** в Winamp с названиями альбомов.

### Что внутри одного плейлиста

- Все аудиофайлы из соответствующей подпапки, отсортированные по имени.
- Поддерживаемые форматы: `mp3`, `m4a`, `mp4`, `flac`, `ogg`, `oga`, `opus`, `wma`, `wav`, `aac`, `ape`, `wv`, `aif`, `aiff`, `mpc`, `tta`.
- Подпапки без аудио пропускаются.
- Если плейлист с таким же именем уже есть — подпапка пропускается (повторный запуск безопасен).

### Безопасность

- Перед изменением создаётся бэкап:
  ```
  %APPDATA%\Winamp\Plugins\ml\playlists\playlists.xml.bak
  ```
- Если что-то пошло не так — закрой Winamp, удали `playlists.xml`, переименуй `playlists.xml.bak` обратно в `playlists.xml`.

## 2. Массовое удаление плейлистов

**Файл:** `WinampRemovePlaylists.bat`

### Как использовать

1. Закрой Winamp.
2. Запусти `WinampRemovePlaylists.bat`.
3. Откроется окно со списком всех плейлистов и галочками.
4. Кнопки: **Отметить все** / **Снять все** / **Удалить отмеченные** / **Отмена**.
5. После подтверждения удаляются и записи в `playlists.xml`, и сами файлы плейлистов `plfXXXX.m3u8`.

> **Важно:** сами музыкальные файлы (mp3 и т. п.) **НЕ удаляются** — стираются только плейлисты Winamp.

Бэкап: `playlists.xml.bak` (можно восстановить вручную).

### Опции командной строки

```bat
WinampRemovePlaylists.bat -All           :: удалить ВСЕ (с GUI-подтверждением)
WinampRemovePlaylists.bat -All -Quiet    :: удалить ВСЕ без всяких окон
```

## 3. Блокировка окна "Доступна новая версия Winamp"

**Файлы:** `WinampBlockUpdates.bat` (включить) / `WinampUnblockUpdates.bat` (откатить)

### Что делает

Запускается с правами администратора (попросит подтверждение UAC) и дописывает в `C:\Windows\System32\drivers\etc\hosts`:

```
# Winamp update block
127.0.0.1 client.winamp.com
127.0.0.1 www.winamp.com
127.0.0.1 winamp.com
```

Делает бэкап `hosts.winampbak` (один раз) и сбрасывает DNS-кэш.

После этого Winamp при запуске не сможет связаться с сервером обновлений и окно "Доступна новая версия" больше не появится.

### Что такое DNS-кэш и зачем `ipconfig /flushdns`?

- **DNS** — система, которая превращает имя сайта (`client.winamp.com`) в IP-адрес.
- Чтобы каждый раз не спрашивать заново, Windows запоминает последние ответы в **DNS-кэше** — небольшой таблице в оперативной памяти службы DNS Client. Живёт от пары минут до нескольких часов.
- Команда `ipconfig /flushdns` просто очищает эту таблицу в памяти.
  - **НЕ** трогает файлы на диске
  - **НЕ** меняет настройки сети
  - **НЕ** требует интернета
  - **НЕ** отправляет никаких данных
  - Windows сама вызывает то же самое при смене Wi-Fi, подключении VPN и т. п.
- **Зачем здесь:** если Winamp до блокировки уже запрашивал `client.winamp.com`, его настоящий IP мог попасть в кэш. Пока кэш не обновится, Windows будет отдавать старый IP и `hosts` будет игнорироваться. `flushdns` убирает старую запись, и блокировка начинает работать сразу.
- Посмотреть текущий кэш: `ipconfig /displaydns` (обычный cmd, без админа).

### Откат

Запусти `WinampUnblockUpdates.bat` (тоже от админа).

## 4. Если не хочется запускать `.bat` — правка `hosts` вручную

Тот же эффект даёт ручная правка файла `hosts`. Никакого DNS flush не требуется — Windows подхватит изменения сама через несколько минут или после перезагрузки.

### Пошагово

1. Нажми **Пуск**, набери "Блокнот" (Notepad).
2. ПКМ по Блокноту → **"Запуск от имени администратора"** → Да.
3. В Блокноте: **Файл → Открыть**.
4. В поле "Имя файла" вставь полный путь:
   ```
   C:\Windows\System32\drivers\etc\hosts
   ```
   (или перейди в эту папку и справа внизу переключи фильтр "Текстовые документы (\*.txt)" на **"Все файлы"**).
5. Нажми **Открыть**.
6. В самый конец файла допиши четыре строки:
   ```
   # Winamp update block
   127.0.0.1 client.winamp.com
   127.0.0.1 www.winamp.com
   127.0.0.1 winamp.com
   ```
7. **Файл → Сохранить** (Ctrl+S). **НЕ "Сохранить как"** — иначе получится `hosts.txt`, а нужен файл **БЕЗ расширения**.
8. Закрой Блокнот.

Готово. При следующем запуске Winamp окна с обновлением не будет. Если Winamp всё-таки ещё видит сервер — перезагрузи комп, и старый адрес из DNS-кэша уйдёт сам.

### Откат

Открой `hosts` тем же способом и удали эти 4 строки.

### Проверка что блокировка работает

Открой обычный cmd и набери:
```
ping client.winamp.com
```
Должно отвечать `127.0.0.1` (или "не удалось найти узел"). Если отвечает какой-то другой IP — кэш ещё не обновился, подожди или перезагрузись.

## 5. Текстовый трек-лист альбома

**Файл:** `MakeTracklist.bat`

### Что делает

Берёт папку с аудиофайлами и создаёт рядом со скриптом текстовый файл `<имя-папки>.txt` со списком песен и сводкой по альбому.

### Как использовать

1. Запусти `MakeTracklist.bat` двойным кликом.
2. В диалоге выбери папку альбома.
3. Рядом со скриптом появится `<имя альбома>.txt`.

### Пример вывода

```
01. Шум воды
02. Лейся дождь
03. Это любовь
...
14. Влеки

---
Битрейт:           940 kbps
Продолжительность: 1:12:08
Размер:            475 МБ (498 336 855 байт)
```

### Детали

- **Названия песен** извлекаются из имён файлов. Поддерживаются разные форматы:
  `01. Artist - Title`, `01.Title`, `01 - Title`, `01_Title`, `01 Title`, просто `Title`.
  Префикс «Artist - » отбрасывается, остаётся только название.
- Если в имени нет номера — нумерация проставляется автоматически (01, 02, ...).
- **Битрейт** берётся из первого файла.
- **Продолжительность** — сумма точных длительностей всех треков (через `System.Media.Duration`), формат `H:MM:SS` или `MM:SS`.
- **Размер** — сумма размеров всех файлов в папке (включая обложку), как в Свойствах Windows.
- Файл сохраняется в **UTF-8 с BOM** — кириллица корректно открывается в Блокноте, Word, на форумах.

## FAQ

**В: У меня Winamp портативный (Portable). Будет работать?**
О: Скрипт ищет `playlists.xml` в:
1. `%APPDATA%\Winamp\Plugins\ml\playlists`
2. `C:\Program Files (x86)\Winamp\Plugins\ml\playlists`
3. `C:\Program Files\Winamp\Plugins\ml\playlists`

Если у тебя нестандартный путь — добавить.

**В: Что если Winamp был запущен во время добавления/удаления?**
О: Скрипт это заметит и попросит закрыть Winamp. Иначе Winamp при выходе перезапишет `playlists.xml` своими данными и изменения исчезнут.

**В: Длительность треков (`seconds`) определяется как?**
О: Через Windows Shell — то же, что показывает Проводник в столбце "Продолжительность". Если для какого-то файла длительность не удалось получить, ставится `-1` — на работу плейлиста не влияет.

**В: PowerShell ругается "выполнение сценариев отключено"?**
О: `.bat`-обёртки запускают PowerShell с флагом `-ExecutionPolicy Bypass` именно для одного запуска, никакие глобальные настройки системы не меняются.
