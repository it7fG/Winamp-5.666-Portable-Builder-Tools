Winamp Tools (работает на любой Windows 7+)
=========================================================

В наборе:
  1) WinampAddAlbums.bat       - массово добавить альбомы в "Списки воспроизведения"
  2) WinampRemovePlaylists.bat - массово удалить плейлисты (все или выборочно)
  3) WinampBlockUpdates.bat    - заблокировать окно "Доступна новая версия"
  4) WinampUnblockUpdates.bat  - откатить блокировку обновлений
  5) MakeTracklist.bat         - делает текстовый трек-лист альбома** — список песен + битрейт, общая длительность и размер папки.
Требования: установленный Winamp (5.x) и PowerShell 2.0+ (есть в любой Windows 7+).
Запуск: двойной клик по .bat-файлу. Никаких установок не нужно.

-----------------------------------------------------------------------
1) Массовое добавление альбомов
-----------------------------------------------------------------------
Сценарий:
  У тебя есть папка, например, D:\Music\Christian, а внутри 30 подпапок -
  каждая подпапка это альбом со своими треками. Скрипт создаст в Winamp
  ТРИДЦАТЬ ОТДЕЛЬНЫХ плейлистов (по одному на альбом). Названия плейлистов
  будут совпадать с именами папок альбомов.

Как использовать:
  - Закрой Winamp (важно: при выходе он перезаписывает playlists.xml).
  - Запусти WinampAddAlbums.bat двойным кликом.
  - В диалоге выбери папку, внутри которой лежат подпапки-альбомы.
  - Дождись окна "Готово" с отчётом.
  - Открой Winamp - новые плейлисты появятся в "Списки воспроизведения".

Что внутри одного плейлиста:
  - Все аудиофайлы из соответствующей подпапки (отсортированные по имени).
  - Поддерживаемые форматы: mp3, m4a, mp4, flac, ogg, oga, opus, wma, wav,
    aac, ape, wv, aif, aiff, mpc, tta.
  - Подпапки без аудио пропускаются.
  - Если плейлист с таким же именем уже есть - подпапка пропускается
    (повторно не добавляется).

Безопасность:
  - Перед изменением создаётся бэкап:
      %APPDATA%\Winamp\Plugins\ml\playlists\playlists.xml.bak
  - Если что-то пошло не так - закрой Winamp, удали playlists.xml,
    переименуй playlists.xml.bak обратно в playlists.xml.

Удаление этих плейлистов: правый клик по плейлисту в Winamp -> Удалить.

-----------------------------------------------------------------------
2) Массовое удаление плейлистов
-----------------------------------------------------------------------
WinampRemovePlaylists.bat:
  - Закрой Winamp перед запуском.
  - Откроется окно со списком всех плейлистов и галочками.
  - Кнопки: "Отметить все", "Снять все", "Удалить отмеченные", "Отмена".
  - После подтверждения удаляются и записи в playlists.xml,
    и сами файлы плейлистов plfXXXX.m3u8.
  - ВАЖНО: сами музыкальные файлы (mp3 и т.п.) НЕ удаляются -
    стираются только плейлисты Winamp.
  - Бэкап: playlists.xml.bak (можно восстановить вручную).

Опции командной строки (необязательно):
  WinampRemovePlaylists.bat -All    - удалить ВСЕ без выбора (через GUI-подтверждение)
  WinampRemovePlaylists.bat -All -Quiet  - удалить ВСЕ без всяких окон (для автоматизации)

3) Блокировка окна "Доступна новая версия Winamp"
-----------------------------------------------------------------------
WinampBlockUpdates.bat:
  - Запускается с правами администратора (попросит подтверждение UAC).
  - Дописывает в C:\Windows\System32\drivers\etc\hosts строки:
        # Winamp update block
        127.0.0.1 client.winamp.com
        127.0.0.1 www.winamp.com
        127.0.0.1 winamp.com
  - Делает бэкап hosts один раз: hosts.winampbak
  - Сбрасывает DNS-кэш командой "ipconfig /flushdns" (см. ниже,
    что это и зачем - ничего страшного, штатная команда Windows).

После этого Winamp при запуске не сможет связаться с сервером обновлений,
и окно "Доступна новая версия" больше не появится.

Откат: запусти WinampUnblockUpdates.bat (тоже от админа).

Что такое DNS-кэш и зачем "ipconfig /flushdns"?
  - DNS - это система, которая превращает имя сайта (client.winamp.com)
    в IP-адрес. Чтобы каждый раз не спрашивать заново, Windows запоминает
    последние ответы в "DNS-кэше" - небольшой таблице в оперативной памяти
    службы DNS Client. Живёт от пары минут до нескольких часов.
  - Команда "ipconfig /flushdns" просто очищает эту таблицу в памяти.
    Она НЕ трогает файлы на диске, НЕ меняет настройки сети, НЕ требует
    интернета и НЕ отправляет никаких данных. Windows сама вызывает то же
    самое каждый раз при смене Wi-Fi, подключении VPN и т.п.
  - Зачем она нужна тут: если Winamp до блокировки уже запрашивал
    client.winamp.com, его настоящий IP мог попасть в кэш. Пока кэш не
    обновится, Windows будет отдавать старый IP и hosts будет
    игнорироваться. flushdns убирает старую запись, и блокировка
    начинает работать сразу.
  - Посмотреть текущий кэш можно командой "ipconfig /displaydns"
    (обычный cmd, без админа).

4) Если не хочется запускать .bat — правка hosts вручную
-----------------------------------------------------------------------
Если боишься запускать .bat файлы, тот же эффект даёт ручная правка
файла hosts. Никакого DNS flush не требуется - в крайнем случае
Windows подхватит изменения сама через несколько минут или после
перезагрузки.

Пошагово:
  1. Нажми Пуск, набери "Блокнот" (Notepad).
  2. ПКМ по Блокноту -> "Запуск от имени администратора" -> Да.
  3. В Блокноте: Файл -> Открыть.
  4. В поле "Имя файла" вставь полный путь:
        C:\Windows\System32\drivers\etc\hosts
     (или перейди в эту папку и справа внизу переключи фильтр
      "Текстовые документы (*.txt)" на "Все файлы").
  5. Нажми "Открыть".
  6. В самый конец файла допиши четыре строки:

        # Winamp update block
        127.0.0.1 client.winamp.com
        127.0.0.1 www.winamp.com
        127.0.0.1 winamp.com

  7. Файл -> Сохранить (Ctrl+S). НЕ "Сохранить как" - иначе
     получится hosts.txt, а нужен файл БЕЗ расширения.
  8. Закрой Блокнот.

Готово. При следующем запуске Winamp окна с обновлением не будет.
Если Winamp всё-таки ещё видит сервер - перезагрузи комп, и старый
адрес из DNS-кэша уйдёт сам.

Откат (вернуть как было):
  Открой hosts тем же способом и удали эти 4 строки.

Проверка что блокировка работает:
  Открой обычный cmd и набери:
        ping client.winamp.com
  Должно отвечать 127.0.0.1 (или "не удалось найти узел"). Если
  отвечает какой-то другой IP - кэш ещё не обновился, подожди или
  перезагрузись.

-------------------------------------------------------------
5) Текстовый трек-лист альбома
--------------------------------------------------------------
Файл: MakeTracklist.bat

Что делает

Берёт папку с аудиофайлами и создаёт рядом со скриптом текстовый файл `<имя-папки>.txt` со списком песен и сводкой по альбому.

Как использовать

1. Запусти `MakeTracklist.bat` двойным кликом.
2. В диалоге выбери папку альбома.
3. Рядом со скриптом появится `<имя альбома>.txt`.

Пример вывода

01. Шум воды
02. Лейся дождь
03. Это любовь
...
14. Влеки

---
Битрейт:           940 kbps
Продолжительность: 1:12:08
Размер:            475 МБ (498 336 855 байт)


Детали

- Названия песен извлекаются из имён файлов. Поддерживаются разные форматы:
  `01. Artist - Title`, `01.Title`, `01 - Title`, `01_Title`, `01 Title`, просто `Title`.
  Префикс «Artist - » отбрасывается, остаётся только название.
- Если в имени нет номера — нумерация проставляется автоматически (01, 02, ...).
- **Битрейт** берётся из первого файла.
- **Продолжительность** — сумма точных длительностей всех треков (через `System.Media.Duration`), формат `H:MM:SS` или `MM:SS`.
- **Размер** — сумма размеров всех файлов в папке (включая обложку), как в Свойствах Windows.
- Файл сохраняется в **UTF-8 с BOM** — кириллица корректно открывается в Блокноте, Word, на форумах.

-----------------------------------------------------------------------
FAQ
-----------------------------------------------------------------------
В: У меня Winamp портативный (Portable). Будет работать?
О: Скрипт ищет playlists.xml в %APPDATA%\Winamp\Plugins\ml\playlists,
   потом в Program Files (x86)\Winamp\... и Program Files\Winamp\...
   Если у тебя нестандартный путь - добавить.

В: Что если Winamp был запущен во время добавления?
О: Скрипт это заметит и попросит закрыть Winamp. Иначе Winamp при выходе
   перезапишет playlists.xml своими данными и новые альбомы исчезнут.

В: Длительность треков (seconds) определяется как?
О: Через Windows Shell (то же, что показывает Проводник в столбце
   "Продолжительность"). Если для какого-то файла длительность не
   удалось получить, ставится -1 - на работу плейлиста не влияет.
